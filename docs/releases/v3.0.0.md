# VEGA v3.0.0 Release Notes

VEGA v3.0.0 delivers the Operator Console and request-local live execution
progress while preserving the v2.13.0 execution, security, and state contracts.

## Added

* Width-aware startup rendering from canonical runtime version, workspace,
  selected model, and readiness state.
* Restrained ANSI colors, no-color output, Unicode symbols, and complete ASCII
  fallbacks.
* Immutable payload-free progress events and fail-soft terminal rendering for
  analysis, planning, validated plans, real steps, confirmation, and outcomes.
* Request-local elapsed time that updates on TTYs and stops on success, failure,
  cancellation, or timeout.
* Exact nullable Ollama input/output/total usage, including final streaming
  usage, plus locale-aware completion summaries.
* Structured session-log request metrics with UTC boundaries, status, duration,
  diagnostic phase durations, and no added prompt/response content.

## Changed

* The old framed ASCII logo and `VEGA SESSION` panel are replaced by
  `VEGA / OPERATOR CONSOLE`.
* The task prompt is one `vega ›` line (`vega >` in ASCII mode).
* Startup no longer displays Internet/network/safety status or session-log
  paths. Those policies and logs still exist and remain available through their
  existing commands.
* The legacy Ollama tuple API remains compatible; a detailed result API now
  distinguishes timeout and carries exact nullable usage.

## Fixed

* Removed the stale contextual `bug_fix` planner route. Bug fixes continue to
  use the controlled coding workflow, and patch tools remain outside automatic
  contextual routing; the production policy snapshot is now warning-free.

## Compatibility and migration

Exact terminal snapshots/parsers require updates. Existing commands, Python
execution result contracts, callback-free callers, plugin API, permission and
routing policies, workflow state, checkpoints, traces, and diagnostics remain
compatible. Legacy banner call shapes are retained without restoring removed
visual fields. No data migration is required.

The unused internal `core.agent_runtime.banner()`/`fallback_banner()` helpers
and standalone execution of `scripts/vega_banner.py` are removed. Use
`ui.startup_screen`, the compatibility `render_banner()` import, or the normal
`scripts/vega.py` entrypoint.

See [`../migrations/v3.0.0.md`](../migrations/v3.0.0.md) for the step-by-step
migration and rollback procedure.

## Security

Progress remains an observer: it receives no tool handlers, arguments, results,
permission authority, prompts, secrets, raw exceptions, or persistence. UI
callback failures cannot cause execution, retry, or result changes.
Token counts are server-returned metadata, never locally estimated. Missing
statistics stay unavailable rather than being logged or displayed as zero.

## Validation

The release is gated by focused UI/progress/compatibility tests, the full test
suite, compileall, identity, production policy consistency, smoke,
documentation, repository hygiene, and the read-only Release Manager.
Focused coverage includes timer lifecycle, formatting, reset, exact/missing
usage, streaming completion, timeout/cancellation, and parallel isolation.
