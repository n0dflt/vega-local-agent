# VEGA v2.12.1 Release Notes

VEGA v2.12.1 stabilizes **Local State Integrity & Recovery** after an independent
clean-environment and security audit of v2.12.0.

## Why a patch release was required

The published test command used `.tmp/pytest-release` while `.tmp` was entirely
ignored. A clean checkout therefore lacked the parent directory and failed
before many tests could start. The audit also found security-relevant recovery
edge cases: substituted content-addressed quarantine evidence was trusted,
report JSON accepted an underspecified object, size-check/read races could exceed
the intended bound, and cleanup failures used an inaccurate scan-limit code.

These are reproducibility and state-integrity defects, not documentation-only
hardening, so the release identity advances to v2.12.1.

## Fixes

* Track only `.tmp/.gitkeep` while continuing to ignore all generated temp data.
* Close raw lock descriptors after stream-construction failure and use no-follow
  opens for lock probes and bounded generated-file reads.
* Read at most the configured byte cap plus one byte, even if a generated file
  changes concurrently.
* Verify an existing quarantine file matches the expected content before any
  repaired source replaces the corrupted file.
* Accept compatible v2.11 and v2.12 diagnostic-report top-level schemas and
  quarantine malformed or underspecified generated reports.
* Report cleanup and quarantine retention failures with repair/quarantine codes.

## CI baseline

Pull requests and pushes to `main` now run on Windows and Linux with Python 3.12,
3.13, and 3.14. The workflow has read-only repository permission, persists no
checkout credential, uses no secrets or external service, pins official actions
to immutable commits, and cannot commit, merge, tag, or publish.

The matrix runs compileall and the full suite. One release-gate job runs focused
state tests, identity, policy consistency, smoke, repository hygiene, and the
built-in Release Manager. Skip reasons are printed explicitly.

## Compatibility and migration

There is no state migration. Valid v2.10/v2.11 traces, v2.11 reports, v2.12
traces/reports/quarantine, public trace APIs, state doctor commands, and existing
manual commands remain compatible. Existing invalid generated reports now fail
closed and are quarantined only after explicit repair.

## Known limitations

Locks remain advisory. Recovery remains bounded rather than forensic. Windows
symlink tests may skip when the account lacks symlink privileges. No background
repair, telemetry, model authority, autonomous execution, or in-product
publication was added.

## Validation

Local Windows/Python 3.12 results were `117 passed, 2 skipped` focused and `996
passed, 5 skipped, 133 subtests` full-suite. Compileall, identity, policy
consistency, smoke, whitespace, and generated-state hygiene passed. All five
skips are explicit symlink-privilege conditions; four predate this patch and the
v2.12 state-lock test uses the same condition. The two collection warnings also
predate this patch. Windows/Linux Python-matrix results are recorded in the
stabilization pull request and GitHub Release.
